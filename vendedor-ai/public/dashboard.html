O Autopilot executa todo esse fluxo automaticamente com um clique. Você pode também rodar cada etapa individualmente pela seção Agentes.

1
🚀
Primeiros Passos
Configure antes de usar
①Vá em Configurações e confirme que Groq API, Gemini API e Notion API estão com status ✓ OK
②Se alguma API mostrar ✗ Erro, a chave não está no arquivo .env da VPS
③Confirme que Instagram Session está ✓ Ativa — se estiver Expirada, rode o scraper com --login no terminal
④Ative o toggle Autopilot em Configurações → Automação
💡 Dica: PM2 Status deve aparecer online em azul. Se não, o servidor precisa ser reiniciado.
2
⚡
Usar o Autopilot
Modo mais simples — recomendado
①Vá ao Command Center (tela inicial)
②Clique no botão ⚡ Iniciar Autopilot — o sistema faz todo o resto automaticamente
③Aguarde — o processo leva 5 a 15 minutos dependendo da quantidade de leads configurada
④Acompanhe em tempo real pela seção Logs ou pelo feed de Atividade Recente
⑤Quando terminar, os leads aparecem no CRM e as DMs ficam na Fila de DMs
💡 Dica: Você pode rodar o Autopilot uma vez por dia. Mais do que isso aumenta o risco de bloqueio no Instagram.
3
👥
CRM — Leads
Gerenciar sua base de contatos
①Todos os leads coletados aparecem aqui com nome, nicho, score e status
②Use os filtros Novos / Score Alto / Contatados para focar nos melhores leads
③O Score vai de 0 a 100 — priorize leads acima de 70
④Clique em 💬 para ver a DM gerada para aquele lead
⑤Clique em ↗ para abrir o perfil do Instagram diretamente
💡 Dica: Score Alto (≥70) significa que o Analyzer identificou um problema real que o seu produto resolve.
4
💬
Fila de DMs
Revisar e enviar mensagens
①Após o Autopilot rodar, as DMs geradas aparecem aqui para revisão
②Leia cada mensagem antes de enviar — o Gemini gera textos personalizados mas é bom revisar
③Clique em ⚡ Enviar para enfileirar uma DM individual para o sender
④Ou clique Enviar Fila Completa para enviar todas de uma vez (respeita limite de 20/dia)
⑤Use Pular para remover uma DM da fila sem enviar
⚠️ Anti-ban: O sistema tem delay automático de 3–8 minutos entre cada DM. Não tente enviar mais de 20 por dia.
5
📊
Analytics
Entender os resultados
①Leads por Dia — volume de coleta nos últimos 7 dias
②Distribuição de Score — qualidade dos leads coletados (quanto mais à direita, melhor)
③Breakdown por Status — quantos leads estão em cada etapa do funil
④Top Nichos — quais nichos o scraper está encontrando mais
💡 Meta: Taxa de Resposta acima de 5% é boa. Acima de 10% é excelente para prospecção fria no Instagram.
6
🤖
Agentes Individuais
Controle avançado do pipeline
①0-Scraper — coleta novos perfis no Instagram por hashtag. Roda em ~2 min
②1-Analyzer — analisa os perfis coletados e atribui score via IA. Roda em ~5 min
③2-Copywriter — gera DMs personalizadas para os leads com score alto. Roda em ~3 min
④9-Notion Sync — sincroniza o CRM local com o Notion. Roda em ~1 min
⑤Para rodar qualquer agente, vá em Agentes e clique em ▶ Rodar
💡 Dica: Use os agentes individuais quando quiser re-executar só uma etapa, como gerar novas DMs sem coletar leads novos.
7
📋
Logs do Sistema
Monitorar o que está acontecendo
①Os logs mostram tudo que os agentes estão fazendo em tempo real
②Use os filtros (0-scraper, 1-analyzer, 10-autopilot) para focar em um agente específico
③Linhas em verde = sucesso. Linhas em vermelho = erro
④Se um agente travar, verifique os logs primeiro — quase sempre mostra o motivo
⑤O botão Limpar apaga os logs visíveis (não apaga os arquivos de log reais)
💡 Dica: Erros de API key aparecem nos logs como 401 Unauthorized — sinal de que a chave expirou ou está errada.
❓
Perguntas Frequentes
Dúvidas comuns de operação
Por que o CRM está vazio?
O Autopilot ainda não foi executado, ou os agentes rodaram com erro. Verifique a seção Logs para ver o que aconteceu. Se as APIs estiverem com ✗ Erro em Configurações, resolva isso primeiro.
O que significa "Instagram Session: Expirada"?
O Playwright precisa de uma sessão de login ativa para acessar o Instagram. A sessão expira periodicamente. Para renovar, execute node agents/0-scraper.js --login no terminal da VPS e faça login manualmente.
Quantas DMs posso enviar por dia?
O limite configurado é 20 DMs/dia com delay de 3–8 minutos entre cada envio. Esse limite existe para proteger a conta do Instagram de ser bloqueada. Não recomendamos aumentar acima de 25.
Os dados sincronizam com o Notion automaticamente?
Sim, se o toggle "Sync Notion" estiver ativo nas Configurações. O processo roda via PM2 em background. Você também pode forçar um sync manual rodando o agente 9-Notion Sync manualmente.
O que fazer se um agente travar ou demorar muito?
Verifique a seção Logs para erros. Se não houver erro visível, acesse a VPS e execute pm2 restart all para reiniciar todos os processos.
